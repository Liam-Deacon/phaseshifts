# Security Policy

## Supported Versions

We support the latest release and the current `master` branch. If you need a
security fix for an older version, please open a discussion or contact us.

## Reporting a Vulnerability

Please report security issues privately using one of the following options:

- GitHub Security Advisories:
  https://github.com/Liam-Deacon/phaseshifts/security/advisories/new
- Email: liam.deacon@diamond.ac.uk

Please include as much detail as possible (affected version, reproduction
steps, impact, and any proposed fixes).

## Response Timeline

We aim to acknowledge security reports within 14 days and will provide status
updates as we investigate.

## Disclosure

We coordinate disclosure with the reporter when possible and publish fixes via
standard release channels.
